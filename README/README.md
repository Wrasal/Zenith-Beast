# Ведение документа
Каждый текстовый файл внутри обсидиана условно назовем **документом**.

## Одобренная документация
Все документы в проекте должны содержать в начале следующую текстовую конструкцию:
~~~mb
---
approved: false
---
~~~

Эти строки обозначают статус проверки.
- false - Не рассмотрено
- true - Одобрено

Все **не рассмотренное** не принято в работу. Это могут быть просто идеи, или документы в разработке.

Учтите, что одобренные документы могут измениться в будущем.

## Комментарии
В документе могут быть комментарии. Оставляйте их там где появились вопросы, на которые не получится ответить в ближайшем будущем. К примеру для решения вашего вопроса о необходимости механики нужно провести массовый тест игры и посмотреть на реакцию игроков.

~~~mb
> [!quote|mark wm-sm] ник автора
> Содержание коментария описывающего проблему и возможные пути решения 
~~~
